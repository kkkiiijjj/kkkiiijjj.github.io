4.对话功能，应用huggingface里的模型实现与用户对话生成语句。

总结

Cursor非常好用

它能做的：

  1. 根据你给出的需求生成代码
  2. 进一步优化代码
  3. 对程序进行调试
  4. 对报错提出解决方案
  5. 补充上传github时需要的文件，如readme、requirements、gitignore

我们要做的：

  1. 提供想要的效果
  2. 运行它写出的代码然后告诉它哪里不对或需要调整、优化
  3. 提供需要的素材，如图片、音频、gif图
  4. 自己建github仓库、运行终端
  5. 检查它写的代码和文字内容，比如我的readme文件内容不完整

知识点

GUI

  1. GUI 是 图形用户界面 的缩写，英文全称是 Graphical User Interface。它是一种人机交互的方式，通过图形化的方式，如窗口、菜单、图标等，来让用户与计算机进行交互。
  2. GUI的组成：
    窗口 (Window)： 是GUI的基本单位，用于显示应用程序的内容。
    菜单 (Menu)： 提供了各种命令和选项，用户可以通过点击菜单来执行操作。
    图标 (Icon)： 用图形化的方式表示应用程序、文件或命令。
    按钮 (Button)： 用户点击按钮来触发特定的操作。
    对话框 (Dialog)： 用于向用户获取信息或显示提示信息。

状态机（State Machine）是一种程序设计的思想，它将程序看作是在不同状态之间转换的系统。

模块化设计

模块化设计是一种软件设计方法，将程序分解成独立的、可重用的模块。
优点：高内聚（相关功能集中在一起）低耦合（模块之间依赖最小化）

事件过滤器

事件过滤器(Event Filter)简单来说，就是一种机制，它允许一个对象拦截并处理其他对象发出的特定类型的事件。
工作流程：
事件传递顺序：

  1. 事件产生（如按键）
  2. 传递给事件过滤器
  3. 如果过滤器返回 True，事件处理结束
  4. 如果过滤器返回 False，事件继续传递给目标对象
    打个比方：
    想象一下，你家门口有一个信箱。通常情况下，邮递员会直接把信投进你的信箱。但是，如果你在信箱上安装了一个过滤器，那么所有的信件在投进信箱之前，都会先经过这个过滤器。过滤器会检查每一封信，决定是否让它进入信箱。

信号槽机制

信号槽机制 是 Qt 框架中用于对象之间通信的一种机制，也是 Qt 的核心特性之一。它可以看作是一种高级的观察者模式。
信号 (Signal): 当一个对象发生了一些变化或者某个事件触发时，它就会发出一个信号，就像广播电台发射信号一样。
槽 (Slot): 另一个对象可以连接到这个信号上，当信号被发出时，连接到这个信号的槽就会被自动调用，就像收音机接收到信号后播放音乐一样。

多线程

多线程 是一种让一个程序同时执行多个任务的技术。
线程 是操作系统能够进行运算调度的最小单位。一个进程可以包含多个线程。
优点：提高并发性、提高资源利用率、改善用户体验

缺点：增加了程序的复杂性及调试的难度

PyQt

PyQt 作为一款强大的 Python GUI 框架，可以用来构建各种各样的桌面应用程序。
PyQt 可以用来做什么？

  • 系统工具: 文件管理器、文本编辑器、系统监控工具等。
  • 数据可视化: 使用 Matplotlib 或其他绘图库，创建交互式图表和数据可视化工具。
  • 科学计算工具: 结合 NumPy、SciPy 等科学计算库，开发数据分析和模拟工具。
  • 游戏: 创建简单的 2D 游戏，甚至一些复杂的 3D 游戏。
  • 教育软件: 开发交互式的学习工具和模拟器。
  • 企业级应用: 构建定制化的业务应用程序，如 CRM、ERP 系统等。
    优点：
  1. 跨平台 PyQt 应用程序可以在 Windows、macOS、Linux 等多个操作系统上运行，无需修改代码。
  2. 组件丰富 PyQt 提供了大量的 UI 组件，包括按钮、文本框、表格、图表等，可以快速构建复杂的界面。
  3. 与 Python 生态系统的集成: PyQt 可以无缝地与 Python 的其他库（如 NumPy、Pandas、Matplotlib）结合，扩展其功能。
  4. 强大的信号与槽机制: PyQt 的信号与槽机制使得不同组件之间的交互变得简单高效。
  5. 活跃的社区: PyQt 有一个庞大的社区，提供丰富的文档、教程和示例代
